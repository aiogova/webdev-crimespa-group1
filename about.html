Tools we used

This application was built using HTML, CSS, and JavaScript. The Foundation CSS framework was used to create a responsive layout, and Vue.js was used as the front-end JavaScript framework to manage application state and user interaction within a single-page application.

For mapping and geospatial features, the Leaflet API was used to display an interactive map of St. Paul, and the Nominatim API was used for forward and reverse geocoding to convert between addresses and latitude/longitude coordinates.

Crime data is accessed through a custom REST API developed in Project 3 of this course. The API is backed by a SQLite3 database containing St. Paul crime incident data obtained from the City of St. Paul open data portal. This REST API was built using Node.js as the server runtime and Express.js as the web framework to handle HTTP requests.

On the client side, the browser’s built-in fetch() API was used to send asynchronous HTTP requests (GET, PUT, and DELETE) to retrieve, add, and remove crime incidents from the database.

Findings

  1. Downtown and University-area neighborhoods have the highest concentration of incidents.
    • Using the neighborhood marker counts (marker popups and table filters), the central neighborhoods (near downtown and the university corridors) consistently showed the largest counts in the top-1000 set. This suggests these areas are local hot-spots for the most recent incidents.
  2. The most common incident types are property-related (theft/shoplifting) and narcotics-related calls.
    • Aggregating incident types in the UI showed a long tail of low-frequency incident types, while property-theft and narcotics-related categories dominated the top counts.
  3. Evening and late-night hours have noticeably higher incident frequency.
    • Sorting and scanning the time column in the visible table indicates a concentration of incidents between ~18:00 and 03:00, suggesting peak time windows for many incident types (especially thefts and assaults).
  4. Certain neighborhoods show elevated rates of violent incidents versus property incidents.
    • By using the row coloring (violent vs property vs other) and neighborhood filters, a few neighborhoods (not the very center) have a higher fraction of violent-crime rows relative to their total incidents. This indicates localized safety differences that might not be obvious from raw counts alone.
  5. Filtering by date range highlights recent shifts (e.g., an uptick in a specific incident type).
    • Using the date-range filter revealed short-term spikes in particular incident types (for example, an increase in narcotics-related incidents over a recent month), which could indicate emerging trends rather than long-term patterns.
  6. Address geocoding limitations create some uncertainty for exact incident location mapping.
